# 拓扑列表列对齐与下边框显示异常

**日期**：2026-07-02
**状态**：已修复
**影响范围**：`TopologyManagement.tsx` 侧边栏拓扑列表

## 问题描述

侧边栏（240px）中的拓扑列表项存在两个视觉问题：
1. **下边框不正常显示**：列表项底部边框显示异常（双线或不一致）
2. **列对齐/长度不够**：列表项内容区域宽度不足，列对齐不一致

## 根因分析

### 问题 1：下边框异常
Ant Design 5 的 `List` 组件默认 `split={true}`，会为父级 `<div>` 添加 `ant-list-split` 类。CSS-in-JS 据此注入 `border-bottom` 规则到每个 `<li>` 上。而 CSS Module `.topologyItem` 已使用 `border: 1px solid var(--topo-border)` 的四边完整边框（卡片风格）。两套边框系统冲突。

**关键细节**：Ant Design v5 使用 CSS-in-JS（`@ant-design/cssinjs`），样式在运行时通过 `<style>` 标签注入。相同特异度下，运行时注入的样式会覆盖构建时 `<link>` 引入的 CSS Module 样式。

### 问题 2：列对齐/宽度不足
- Ant Design `List.Item` + `size="small"` 通过 CSS-in-JS 注入 `padding: 8px 0`（仅上下内边距，无左右内边距）
- 覆盖了 CSS Module 的 `padding: 12px`
- 导致列表项内容贴边、列对齐不一致

### 问题 3：浏览器缓存
用户持续看到旧 CSS hash（`sm43p`）而非新构建的 hash（`cmaad`）。根因：Nginx 原使用 `Cache-Control: no-cache`（允许浏览器缓存，需重新验证），在 Docker 重建后仍可能返回 304 响应，导致浏览器继续使用旧的 `index.html`（其引用旧的 CSS/JS 哈希文件）。

## 修复方案

### 修复 1：消除 Ant Design 边框冲突
**文件**：`TopologyManagement.tsx` L411
**改动**：`<List split={false} ...>`
**原理**：移除 `ant-list-split` 类，Ant Design CSS-in-JS 的 `border-bottom` 规则不再匹配

### 修复 2：强制覆盖 padding
**文件**：`TopologyManagement.module.css` L84
**改动**：`padding: 12px !important;`
**原理**：`!important` 是 CSS 最高优先级，绝对压制 CSS-in-JS 动态注入的任何 padding 规则

### 修复 3：禁止 HTML 缓存
**文件**：`项目文件/nginx/includes/shared.conf` + `项目文件/frontend/nginx.conf`
**改动**：`Cache-Control: no-cache` → `no-store`
**原理**：`no-store` 彻底禁止浏览器缓存 `index.html`，确保每次加载获取最新版本（引用最新 CSS/JS 哈希文件）
