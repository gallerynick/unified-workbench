# 推流端音频源问题（不添加音频源却有声音 / 添加了音频源却无声音）

## 问题分类
WebRTC / MediaStream / 推流

## 触发场景
StreamStudio 推流页面：
- 用户只添加了画面源（屏幕/摄像头），没有添加任何音频源，但观看端依然能听到声音
- 用户显式添加了麦克风源，但观看端听不到声音

## 根因

### 场景1：不添加音频却有声
`getUserMedia({ video: true, audio: true })` 和 `getDisplayMedia({ video: true, audio: true })` 在获取视频流时**自动包含了音频轨道**。

```ts
// BUG：audio: true 导致视频源自带音频
const stream = await navigator.mediaDevices.getUserMedia({ video: true, audio: true });
```

### 场景2：添加了音频却无声
原代码 `streamRef.current = videoSource.stream!` 只传入了视频源的 MediaStream 给 MediaRecorder。音频源的 MediaStream 从未被合并到录制流中，导致用户虽然添加了麦克风源，但推流数据中不含音频轨道。

## 修复方案

### 修复1：视频源不自动请求音频
```ts
const stream = await navigator.mediaDevices.getUserMedia({ video: true, audio: false });
```

### 修复2：合成流（合并视频和音频轨道）
```ts
const combined = new MediaStream();
// 从视频源提取视频轨道
videoSource.stream.getVideoTracks().forEach(t => combined.addTrack(t));
// 从所有音频源提取音频轨道
allAudioSources.forEach(src => {
  src.stream.getAudioTracks().forEach(t => combined.addTrack(t));
});
mediaRecorder = new MediaRecorder(combined, { mimeType });
```

### 修复3：动态编码器选择
根据音频轨道是否存在选择编码器：
```ts
const hasAudio = combined.getAudioTracks().length > 0;
const mimeType = hasAudio
  ? 'video/webm;codecs=vp8,opus'
  : 'video/webm;codecs=vp8';
```

## 易错点
- `getDisplayMedia` 不支持 `audio: false` 参数——需要先获取 `{ video: true, audio: true }` 再手动停掉音频轨道
- 合成流时注意轨道类型区分：`getVideoTracks()` / `getAudioTracks()`
- 编码器 MIME type 必须与实际轨道匹配，否则 `MediaRecorder.isTypeSupported()` 返回 false
