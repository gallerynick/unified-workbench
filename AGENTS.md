# 统一工作台 (Unified Workbench) — Agent 指引

> 本仓库是统一工作台的完整代码仓库，包含前后端代码、配置文件和开发文档。

## 仓库结构

```
/
├── 项目基定/                  # 【只读】项目规格，不可修改
│   ├── 开发基准文档.md         # 技术架构、功能模块、数据库设计、安全基准
│   ├── 文件目录及说明.txt       # 目录结构说明
│   └── 项目开发日志格式.md     # 日志模板，每次开发后必须按此格式写日志
├── 项目开发日志/              # 【读写】仅放日志，不放代码
├── 项目文件/                  # 【读写】开发产物
│   ├── backend/               # 后端代码 (FastAPI + SQLAlchemy)
│   ├── frontend/              # 前端代码 (React + TypeScript)
│   ├── nginx/                 # Nginx 配置
│   ├── docker-compose.yml     # Docker Compose 配置
│   ├── .env.example           # 环境变量模板
│   ├── README.md              # 项目说明
│   └── start.bat              # Windows 启动脚本
└── AGENTS.md                  # 本文件
```

## 开发总则（强制）

1. **基定优先** — 开发前必须先读《项目基定/开发基准文档.md》，所有功能、设计、技术选型以它为准。
2. **版本管理** — 使用语义化版本号（如 `v1.0.0`），通过 Git tag 管理版本。不需要创建版本目录。
3. **日志强制** — 开发前读最近日志了解进度；开发后按 `项目基定/项目开发日志格式.md` 的格式更新日志。
4. **改动透明** — 日志必须列出所有新增、修改、删除的文件。
5. **遇疑即问** — 与基定冲突、版本不明、需求不清时，停止并询问，不得自行假设。

## 技术栈速查

完整规格见 `项目基定/开发基准文档.md`，以下是代理最常需要的：

| 层级 | 技术 |
|------|------|
| 后端 | Python 3.11+ / FastAPI / SQLAlchemy 2.0 / Alembic |
| 前端 | React 18 / TypeScript 5 / Ant Design 5 / Tiptap(富文本) |
| 数据库 | PostgreSQL 15+ |
| 缓存/队列 | Redis 7 / Celery |
| 实时推送 | WebSocket |
| 部署 | Windows + Docker Desktop + Docker Compose |
| 文档导出 | python-docx(Word) / WeasyPrint(PDF) / openpyxl(Excel) |

**后端目录结构（规划）：**
```

├── api/        # 路由
├── core/       # 配置、安全、依赖
├── models/     # 数据库模型
```

## 关键安全约束（不可违反）

- 密码：bcrypt 哈希，≥8位含字母+数字
- 密钥分区：AES-256-GCM 加密，强制私有
- 查看密钥值：二次输入登录密码验证
- 会话：JWT + 刷新机制 + 闲置超时登出
- 传输：全站 HTTPS
- 审计日志：只增不改不删

## 代码规范

- 后端：PEP8 + ruff + black + mypy
- 前端：ESLint + Prettier + 严格 TypeScript
- API：RESTful，统一响应 `{ "code": 0, "msg": "", "data": {} }`
- 所有 API 用 Pydantic 模型，自动生成 OpenAPI 文档
