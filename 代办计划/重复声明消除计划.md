# 重复声明消除计划（P0 + P1）

## P0-1：可见性 `public/private/restricted` 统一为枚举

### 现状
7 个模型文件独立定义 `visibility: Mapped[str] = mapped_column(String(20), default="private")`：
- `models/record.py:62` — `server_default="private"`
- `models/content.py:29` — `server_default="private"`
- `models/file.py:35` — `server_default="private"`
- `models/folder.py:31` — `server_default="private"`
- `models/form.py:31` — `server_default="private"`
- `models/vote.py:38` — `server_default="private"`
- `models/topology.py:35` — `server_default="private"`

6 个 Schema 文件独立定义 visibility 字段：
- `schemas/content.py:14` — `visibility: str = "private"`
- `schemas/record.py:21` — `visibility: str = "private"`
- `schemas/form.py:25` — `Literal["public", "private", "restricted"]`
- `schemas/topology.py:24` — `Literal["public", "private", "restricted"]`
- `schemas/file.py:39` — `visibility: str = "private"`
- `schemas/vote.py:18` — `Literal["public", "private", "restricted"]`

### 修改方案

**步骤1**：在 `backend/app/core/` 新建 `visibility.py`（Visibility 枚举）：
```python
class Visibility(str, enum.Enum):
    PUBLIC = "public"
    PRIVATE = "private"
    RESTRICTED = "restricted"
```

**步骤2**：7 个模型文件：将 `String(20), server_default="private"` 替换为 `String(20), server_default=Visibility.PRIVATE`（导入 Visibility）

**步骤3**：6 个 Schema 文件：将 `visibility: str = "private"` 替换为 `visibility: Visibility = Visibility.PRIVATE`，将 `Literal["public", "private", "restricted"]` 替换为 `Visibility`

**步骤4**：后端权限/可见性检查文件（`core/permissions.py`、`services/visibility.py`、`services/record.py`）
将 `"public"`/`"private"`/`"restricted"` 字面量替换为 `Visibility.PUBLIC`/`Visibility.PRIVATE`/`Visibility.RESTRICTED`

**步骤5**：Alembic 迁移 — 无。数据库列定义不变，仅在 Python 层面统一。

### 测试验证
| 检查项 | 方法 |
|--------|------|
| Python 导入 | `python -c "from app.core.visibility import Visibility; print(Visibility.PRIVATE)"` |
| 模型正常 | `python -c "from app.models import *; print('OK')"` |
| Alembic 无变更 | `alembic check` — 应输出 "No new upgrade operations detected" |
| API 创建记录 | `curl POST /api/v1/records` 不传 visibility → 验证默认值为 `"private"` |
| API 更新可见性 | `curl PUT` 传 `visibility: "public"` → 验证返回正确 |
| 前端显示 | 项目列表/内容列表等页面可见性 Tag 正常显示 |

## P0-2：Schema 字段复用 Model 枚举类型

### 现状
6 个 Response Schema 文件将枚举字段声明为 `str` 而非枚举类型：
| Schema 文件 | 字段 | 行号 |
|------------|------|------|
| `schemas/user.py` | `role` | 第 29 行 |
| `schemas/user.py` | `status` | 第 30 行 |
| `schemas/budget.py` | `period` | 第 60 行 |
| `schemas/budget.py` | `status` | 第 61 行 |
| `schemas/subscription.py` | `billing_cycle` | 第 60 行 |
| `schemas/subscription.py` | `status` | 第 62 行 |
| `schemas/calendar_event.py` | `repeat` | 第 59 行 |
| `schemas/stream_room.py` | `mode` | 第 60 行 |
| `schemas/stream_room.py` | `room_type` | 第 61 行 |

### 修改方案
每个 Schema 文件将 `field: str` 替换为 `field: <EnumType>`：
- `schemas/user.py:29` → `role: UserRole`
- `schemas/user.py:30` → `status: UserStatus`
- `schemas/budget.py:60` → `period: BudgetPeriod`
- `schemas/budget.py:61` → `status: BudgetStatus`
- `schemas/subscription.py:60` → `billing_cycle: BillingCycle`
- `schemas/subscription.py:62` → `status: SubscriptionStatus`
- `schemas/calendar_event.py:59` → `repeat: EventRepeat`
- `schemas/stream_room.py:60` → `mode: StreamRoomMode`
- `schemas/stream_room.py:61` → `room_type: StreamRoomType`
