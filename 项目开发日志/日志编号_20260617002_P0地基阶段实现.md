### 开发内容详述

**数据库模型**：按基定 §7 创建 user、tag、user_tag 三张表。User 使用 UUID 主键，role 和 status 使用 StrEnum 枚举。所有时间戳使用 UTC。

**认证系统**：使用 python-jose 实现 JWT（access token 30 分钟，refresh token 7 天），bcrypt 实现密码哈希。密码策略：≥8 位，必须包含字母和数字。无开放注册，仅管理员通过用户管理 API 创建账号。

**权限框架**：RBAC 两级角色（admin/member）。`require_admin` 依赖阻止非管理员访问。`check_visibility` 实现三态可见性模型（public/private/restricted），为 P1 文件/内容管理做准备。

**用户管理**：管理员可创建、编辑、禁用用户（软删除）。支持按用户名/昵称搜索，分页查询。标签通过 user_tag 关联表管理。

**前端**：登录页使用 Ant Design Form，居中卡片布局。路由守卫 AuthGuard 检查令牌有效性，未登录重定向到 /login。用户管理页面在"系统设置"子菜单下，包含表格、搜索、新建/编辑/禁用功能。

**种子脚本**：应用启动时检查是否存在管理员，不存在则从 .env 读取凭据创建。幂等设计，重复启动不会创建重复管理员。

**关键决策**：
- 删除 `POST /auth/register`，统一使用 `POST /api/v1/users` 创建用户（避免重复端点）
- 数据库引擎改为惰性初始化，解决测试环境注入 SQLite 的需求
- 前端使用原生 fetch 而非 axios（20 人团队无需额外依赖）
- 测试使用 SQLite 内存数据库（aiosqlite），无需 PostgreSQL 即可运行

### 遇到的问题与解决
- 问题 1：conftest.py 设置 SQLite URL 但 config.py validator 强制要求 postgresql+asyncpg:// 前缀
  - 解决方案：conftest.py 使用合法 PostgreSQL URL 通过验证，同时创建独立 SQLite 引擎用于测试
- 问题 2：database.py 模块级创建引擎，导入时就尝试连接 PostgreSQL
  - 解决方案：重构为惰性初始化模式（`_get_engine()`、`_get_session_factory()`）
- 问题 3：JWT payload 中 user_id 是字符串，但 User.id 是 UUID 列
  - 解决方案：在 deps.py 和 auth.py 中添加 `uuid.UUID(user_id)` 转换
- 问题 4：UserResponse 的 tags 关系在 async 上下文中惰性加载失败
  - 解决方案：添加 `model_validator(mode="before")` 处理惰性加载
- 问题 5：前端 exactOptionalPropertyTypes 导致多处 TypeScript 编译错误
  - 解决方案：使用 `?? []`、`?? ''`、`null` 替代 `undefined`
- 遗留问题 / 待办：无
