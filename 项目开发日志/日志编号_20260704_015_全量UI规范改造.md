### 开发内容详述

#### Wave 1：Token 基础设施
1. **ThemeContext.tsx**：将 getAntdThemeConfig(isDark) 从仅含 colorPrimary 扩展为完整配置：
   - token 全局层：fontFamily, fontSize, controlHeight, borderRadius, colorBorder, colorBgContainer, colorText, colorTextSecondary, colorBgElevated, boxShadow 等 20 个 token
   - components 组件层：31 个组件（Table, Button, Modal, Tag, Badge, Avatar, Card, Select, Switch, Checkbox, Radio, Tree, Segmented, List, Dropdown, Tooltip, Popover, Progress, Collapse, Descriptions, Drawer, Upload, Steps, Timeline, Result, Statistic, Slider, Rate 等），每个组件含 2-8 个 token，所有颜色通过 isDark 三元表达式返回浅/深色值
2. **global.css**：body 添加 `font-family: var(--font-body)`（系统字体）和 `font-size: var(--text-body-size)`（17px），深色模式通过 `[data-theme="dark"] input, button, select { border-color: var(--border-primary); }` 覆盖 Ant Design 默认边框
3. **styles/theme.css**：删除死代码（grep 确认无任何文件引用），该文件 `--color-primary: #1677ff` 与 token.css 的 `--color-primary: #0066cc` 冲突

#### Wave 2：硬编码颜色替换（9 文件）
将所有 `#000`, `#fff`, `#1677ff`, `#6366f1`, `#52c41a`, `#faad14`, `#ff4d4f`, `#868e96` 等硬编码颜色替换为 CSS 变量。ContentEditor.tsx 保留 40 色业务调色板（用户数据，非 UI 样式）。

#### Wave 3：CSS Module 间距 token 化（27 文件）
将 CSS 文件中的硬编码间距值替换为 token.css 中的 CSS 变量：4px→var(--spacing-xxs), 8px→var(--spacing-xs), 12px→var(--spacing-sm), 16px→var(--spacing-card-gap), 24px→var(--spacing-lg), 32px→var(--spacing-xl), 48px→var(--spacing-xxl)。StreamWatch.module.css 为新建文件。

#### Wave 4：内联 style token 化（25 文件，脚本批量处理）
使用 tokenize_inline_styles.py 脚本批量替换 129 处内联 style magic number。替换规则：数字 → 字符串形式的 CSS 变量引用（如 `marginBottom: 8`→`marginBottom: 'var(--spacing-xs)'`）。保留 margin: 0 和已 token 化的值不变。脚本完成后手动修复 Welcome.tsx padding: 100→padding: '40px'。

#### docker-compose.yml
添加 `name: unified-workbench` 以兼容 Docker Compose v2.24+（不再自动使用目录名作为项目名）。

### 遇到的问题与解决

#### 问题 1：后台 agent 任务频繁超时
- **原因**：Wave 4 将 25 个 TSX 文件拆为 6 批并行 agent 任务，每批 30min 超时限制下无法完成
- **解决方案**：改用 Python 脚本 tokenize_inline_styles.py 一次性处理所有 25 个文件，129 处替换秒级完成
- **经验**：大批量简单文件修改应优先脚本而非 agent 逐个编辑

#### 问题 2：Docker Compose v2.24+ 项目名要求
- **原因**：Docker Compose v2.24+ 不再自动使用目录名作为项目名，`docker compose up` 报错 "project name must not be empty"
- **解决方案**：在 docker-compose.yml 顶部添加 `name: unified-workbench`，与 start.sh 中 `-p unified-workbench` 一致
- **遗留问题**：无

#### 问题 3：Token 化后 tsc 类型错误
- **原因**：部分 TSX 文件内联 style 从数字类型变为字符串类型（如 `marginBottom: 8`→`marginBottom: 'var(--spacing-xs)'`），TypeScript 不允许 string 类型的 margin
- **解决方案**：脚本自动处理，字符串形式的 CSS 变量值在 React 的 CSSProperties 中通过 string & {} 类型兼容

#### 问题 4：ContentEditor.tsx 40 色业务调色板
- **原因**：ContentEditor 包含用户可见的 40 色调色板数据（业务数据，非 UI 样式）
- **解决方案**：明确区分业务数据和 UI 颜色，仅替换 UI 颜色，保留业务调色板原值
