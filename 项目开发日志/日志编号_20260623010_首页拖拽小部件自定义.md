### 开发内容详述
1. **类型定义**：在 `types.ts` 中定义 `WidgetType`、`WidgetItem`、`WidgetLayout`，提供默认布局 `DEFAULT_WIDGET_LAYOUT` 和元数据 `WIDGET_META`。
2. **Widget 组件拆分**：将原有首页的 4 个统计卡片提取为 `StatsWidget`，并新增 5 个功能 widget（日程日历、最新公告、通知提醒、待办事项、快捷入口）。每个 widget 使用模拟数据展示，未来可接入真实 API。
3. **拖拽框架**：使用已安装的 `@dnd-kit/core` 和 `@dnd-kit/sortable` 实现网格拖拽排序。`SortableWidget` 作为通用拖拽容器，提供拖拽手柄和隐藏按钮。
4. **显示/隐藏控制**：`WidgetSelectorModal` 提供开关列表，用户可控制各 widget 的显隐。
5. **持久化**：布局（顺序与显隐状态）通过 `localStorage` 的 `home-widget-layout` 键保存与恢复；加载时会合并旧数据与默认配置，防止新增 widget 时丢失。
6. **样式**：`Home.module.css` 提供响应式网格布局（移动端单列、桌面端双列），支持 `prefers-color-scheme: dark` 和 `[data-theme='dark']` 两种暗色模式。
7. **首页重写**：`Home.tsx` 整合以上能力，保留 `useCustomization` 的欢迎语与描述，新增“自定义布局”按钮。

### 遇到的问题与解决
- **问题**：`CalendarWidget` 直接依赖 `dayjs`，但 `dayjs` 不是显式依赖项。
- **解决方案**：移除 `dayjs` 导入，改用原生 `Date` 工具函数生成日期字符串。
- **问题**：Ant Design 5 的 `Tag size="small"`、`Text type` 条件 undefined、`TaskOutlined` 不存在等 TypeScript 严格模式报错。
- **解决方案**：移除不支持的 props；使用 props 展开条件语法 `{...(item.read ? { type: 'secondary' } : {})}` 规避 `exactOptionalPropertyTypes`；替换为 `CheckSquareOutlined`。
- **问题**：CSS Module 路径因 `Home.tsx` 与 `home/` 子目录的关系导致引用错误。
- **解决方案**：统一将 `Home.module.css` 放在 `home/` 目录下，并在 `Home.tsx` 中通过 `./home/Home.module.css` 引用。
