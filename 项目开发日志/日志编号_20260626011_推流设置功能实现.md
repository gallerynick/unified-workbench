### 开发内容详述

**后端：**
- `stream.py` 服务层：基于 `system_config` 的 `get_config`/`update_config` 存储推流配置和用户密钥。配置以 `stream_config` 为键存储在系统配置表中，用户密钥以 `stream_keys` 为键存储（dict 结构，key 为 user UUID 字符串，value 为密钥）。适配了项目中 User.id 为 UUID 类型。
- `stream.py` API 层：4 个端点 — `GET /config`（获取配置）、`PUT /config`（更新配置）、`GET /key`（获取当前用户密钥和推流地址）、`POST /key/reset`（重置密钥）。所有端点均需认证。
- 路由注册到 `router.py`，挂载在 `/stream` 前缀下。

**前端：**
- `stream.ts` API 封装：4 个函数对应后端 4 个端点，返回类型化响应。
- `StreamSettings.tsx` 页面：包含服务器设置卡片（地址、端口、认证开关）、编码参数卡片（分辨率下拉、帧率下拉、码率范围输入、默认码率滑块）、推流密钥卡片（密码框显示密钥、复制按钮、重置按钮、推流地址显示）。使用 `Form.useWatch` 实现码率滑块范围与输入联动。
- CSS 模块因项目启用了 `exactOptionalPropertyTypes`，所有 `styles.xxx` 引用均添加了 `?? ''` 空字符串兜底。
- 路由添加到 `router.tsx` 的 `stream/settings` 路径。
- 系统设置子菜单（仅管理员可见）新增「推流配置」菜单项，使用 `VideoCameraOutlined` 图标。
- `routeTitles.ts` 新增 `/settings/stream` → `推流配置` 映射。

### 遇到的问题与解决
- 问题：原任务描述中 `user_id` 使用 `int` 类型，但项目 User 模型的 `id` 是 `uuid.UUID`。
  解决方案：将 `get_user_stream_key` 和 `reset_user_stream_key` 的参数类型改为 `uuid.UUID`，字典键使用 `str(user_id)`。
- 问题：CSS 模块类型与 Ant Design 的 `exactOptionalPropertyTypes` 不兼容。
  解决方案：所有 `className={styles.xxx}` 改为 `className={styles.xxx ?? ''}`。
- 问题：未使用的 `config` 状态变量导致 TS 编译错误。
  解决方案：移除 `config` 状态，配置值直接通过 `form.setFieldsValue` 管理。
