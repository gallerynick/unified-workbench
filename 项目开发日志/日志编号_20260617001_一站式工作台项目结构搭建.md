### 开发内容详述

**技术决策：**

1. **后端异步架构**：选择 SQLAlchemy 2.0 异步模式 + asyncpg 驱动，配合 FastAPI 的 async/await，确保高并发场景下的性能。数据库连接池配置 pool_size=20，max_overflow=10。

2. **前端版本锁定**：React 锁定 18.3.x（符合基定要求），TypeScript 锁定 5.6.x，Vite 使用 6.0。避免使用 React 19（虽为最新但偏离基定）。

3. **ESLint 9 扁平配置**：采用新的 eslint.config.js 格式（非旧版 .eslintrc），配合 typescript-eslint strict preset，确保代码质量。

4. **Docker Compose 六服务架构**：
   - db (PostgreSQL 15) + redis (Redis 7)：数据层
   - backend (FastAPI) + celery：业务层
   - frontend (React) + nginx：展示层
   - 健康检查确保服务依赖顺序

5. **统一响应格式**：所有 API 返回 `{ "code": 0, "msg": "", "data": {} }`，通过 `UnifiedResponse` 泛型模型实现。

**关键实现：**

- `backend/app/core/config.py`：使用 pydantic-settings，从 .env 文件加载配置，含 DATABASE_URL 格式验证
- `backend/app/core/database.py`：异步引擎 + 异步会话工厂 + 依赖注入 `get_db()`
- `backend/alembic/env.py`：异步迁移环境，从 config 模块获取 DATABASE_URL
- `frontend/src/layouts/MainLayout.tsx`：Ant Design Layout + 可折叠 Sider + 8 个导航菜单项
- `frontend/src/router.tsx`：React Router 6 路由配置，含 404 兜底
