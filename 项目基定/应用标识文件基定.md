# 应用标识文件基定（Marker File Specification）

## 一、概述

应用标识文件是一种**仓库级身份验证机制**，用于防止自动更新系统误将其他应用的仓库当作本应用的更新源，导致更新失败或系统损坏。

## 二、标识文件规格

### 2.1 文件命名

- 文件名：`.unified-workbench`
- 位置：仓库根目录
- 格式：JSON
- 编码：UTF-8（无 BOM）

### 2.2 文件内容

```json
{
  "app_id": "unified-workbench",
  "min_version": "0.1.0"
}
```

### 2.3 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `app_id` | string | ✅ | 应用唯一标识符，必须与后端代码中 `APP_ID` 常量一致 |
| `min_version` | string | ✅ | 最低支持版本号（语义化版本），低于此版本的实例不应尝试更新 |

### 2.4 命名规则

- 文件名格式：`.<app_id>`（以应用标识符命名，前缀加点）
- 例如：app_id 为 `unified-workbench` → 文件名为 `.unified-workbench`

## 三、验证流程

### 3.1 更新检查时的验证

```
用户点击"检查更新"
    ↓
后端读取配置的仓库地址（默认 gallerynick/unified-workbench）
    ↓
后端请求 https://raw.githubusercontent.com/{owner}/{repo}/main/.unified-workbench
    ↓
├─ 文件不存在 → 返回错误："该仓库不是一站式工作台应用（缺少标识文件）"
├─ JSON 解析失败 → 返回错误："标识文件格式错误"
├─ app_id 不匹配 → 返回错误："该仓库不是一站式工作台应用（app_id: xxx）"
└─ 验证通过 → 继续检查版本
    ↓
请求 GitHub Releases API 获取最新版本
    ↓
比较远程版本与当前版本
    ↓
├─ 远程版本 > 当前版本 → 显示"有更新可用"
└─ 远程版本 <= 当前版本 → 显示"已是最新版本"
```

### 3.2 执行更新时的验证

```
用户点击"立即更新"
    ↓
再次执行上述标识文件验证（防止配置被篡改）
    ↓
验证通过 → 执行 git pull + docker compose up -d --build
```

### 3.3 保存仓库地址时的验证

```
用户输入新仓库地址并点击"保存"
    ↓
验证格式（必须为 owner/repo）
    ↓
请求远程标识文件并验证 app_id
    ↓
├─ 验证失败 → 返回错误，不保存
└─ 验证通过 → 保存到数据库
```

## 四、实现细节

### 4.1 后端常量

```python
# app/services/updater.py

APP_ID = "unified-workbench"  # 必须与标识文件中的 app_id 一致
DEFAULT_GITHUB_REPO = "gallerynick/unified-workbench"  # 默认仓库地址
```

### 4.2 验证函数

```python
async def validate_repo(repo: str) -> dict:
    """验证仓库是否为本应用仓库"""
    async with httpx.AsyncClient() as client:
        # 构造标识文件 URL
        url = f"https://raw.githubusercontent.com/{repo}/main/.unified-workbench"
        
        # 请求标识文件
        resp = await client.get(url, timeout=10)
        if resp.status_code != 200:
            return {"valid": False, "error": "该仓库不是一站式工作台应用（缺少标识文件）"}
        
        # 解析 JSON
        try:
            marker = resp.json()
        except json.JSONDecodeError:
            return {"valid": False, "error": "标识文件格式错误"}
        
        # 验证 app_id
        if marker.get("app_id") != APP_ID:
            return {
                "valid": False,
                "error": f"该仓库不是一站式工作台应用（app_id: {marker.get('app_id')})"
            }
        
        return {"valid": True, "app_id": marker["app_id"]}
```

### 4.3 版本比较函数

```python
def version_compare(v1: str, v2: str) -> int:
    """比较版本号
    返回值：
      1  → v1 > v2
      0  → v1 == v2
     -1  → v1 < v2
    """
    parts1 = [int(x) for x in v1.split(".")]
    parts2 = [int(x) for x in v2.split(".")]
    for a, b in zip(parts1, parts2):
        if a > b: return 1
        if a < b: return -1
    return len(parts1) - len(parts2)
```

## 五、安全考虑

### 5.1 防止误更新

- **标识文件验证**：确保远程仓库是本应用的仓库
- **版本比较**：只允许升级，不允许降级
- **二次验证**：执行更新前再次验证标识文件

### 5.2 防止仓库地址篡改

- 保存仓库地址时验证标识文件
- 验证失败不保存，返回明确错误信息

### 5.3 错误信息

| 场景 | 错误信息 |
|------|---------|
| 标识文件不存在 | "该仓库不是一站式工作台应用（缺少标识文件）" |
| JSON 解析失败 | "标识文件格式错误" |
| app_id 不匹配 | "该仓库不是一站式工作台应用（app_id: xxx）" |
| GitHub API 无法访问 | "无法获取远程版本信息" |

## 六、前端交互

### 6.1 系统设置页面

- **更新仓库配置**：输入框 + 保存按钮
- **系统更新**：当前版本、仓库地址、检查更新按钮、更新按钮

### 6.2 用户操作流程

1. 进入 **系统设置与管理 → 系统更新**
2. 查看当前仓库地址（默认 `gallerynick/unified-workbench`）
3. 如需修改仓库地址：
   - 输入新地址（格式：`owner/repo`）
   - 点击"保存"
   - 系统自动验证标识文件
   - 验证通过则保存，失败则提示错误
4. 点击"检查更新"
5. 有更新则显示"立即更新"按钮
6. 点击"立即更新" → 确认 → 执行更新

## 七、扩展性

### 7.1 未来可扩展字段

```json
{
  "app_id": "unified-workbench",
  "min_version": "0.1.0",
  "update_channel": "stable",
  "required_components": ["docker", "git"],
  "breaking_changes": ["0.2.0", "0.3.0"]
}
```

### 7.2 多应用支持

如果未来有多个应用使用同一更新系统，只需：
- 每个应用有唯一的 `app_id`
- 每个应用有对应的标识文件 `.<app_id>`
- 后端 `APP_ID` 常量与标识文件匹配
