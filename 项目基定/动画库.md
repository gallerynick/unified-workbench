# 动画库

> 本文档定义项目中所有动画效果，供前端开发时统一引用。每种动画包含 keyframes 定义、使用方式、参数说明和注意事项（避坑）。

---

## 1 入场动画

### 1.1 菜单项飞入（menuItemFlyIn）

**效果**：元素从左侧模糊飞入，逐渐清晰并滑入最终位置。

**适用场景**：侧边栏菜单项首次加载、列表项逐条入场。

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 统一时长 | `300ms` | 每个元素的动画时长 |
| 交错延迟 | `40ms` | 相邻元素的播放间隔 |
| 缓动曲线 | `ease-out` | 先快后慢，入场自然 |
| 初始模糊 | `6px` | `filter: blur(6px)` |
| 水平偏移 | `-16px` | `transform: translateX(-16px)`，向左偏移 |

**keyframes**：

```css
@keyframes menuItemFlyIn {
  0%   { opacity: 0; filter: blur(6px); transform: translateX(-16px); }
  100% { opacity: 1; filter: blur(0);   transform: translateX(0);      }
}
```

**使用方式（CSS Modules + :global）**：

```css
/* 初始状态：隐藏 + 模糊 + 偏移 */
:global(.your-container .your-item) {
  opacity: 0;
  filter: blur(6px);
  transform: translateX(-16px);
}

/* 触发类添加后播放动画 */
:global(.your-entered .your-item) {
  animation: menuItemFlyIn 400ms ease-out forwards;
}

/* 交错延迟 */
:global(.your-entered .your-item:nth-child(1))  { animation-delay: 0ms; }
:global(.your-entered .your-item:nth-child(2))  { animation-delay: 60ms; }
:global(.your-entered .your-item:nth-child(3))  { animation-delay: 120ms; }
/* ... 继续到所需数量 */
```

**JS 触发（React）**：

```tsx
const [entered, setEntered] = useState(false);
useEffect(() => {
  const timer = setTimeout(() => setEntered(true), 100);
  return () => clearTimeout(timer);
}, []);

<div className={entered ? 'your-entered' : ''}>
  {/* items */}
</div>
```

**注意事项**：
- 需要 JS 配合：组件挂载后等待一帧（~100ms）再添加触发类，确保 DOM 已渲染
- `forwards` 填充模式确保动画结束后元素保持最终状态，不回弹
- 交错延迟 = 统一时长 / 元素数量 × 调节系数，当前 40ms 适用于 22~25 个元素
- 如果同时有 `.ant-menu-item` 和 `.ant-menu-submenu-title`，两者都需要在选择器中覆盖
- **首次挂载只触发一次**：不要复用 `entered` 状态到路由切换等场景

### 1.2 列表逐条浮入（widgetFadeUp）

**效果**：列表项从下方淡入，不含位移（避免与拖拽库的 transform 冲突）。

**适用场景**：首页小组件、卡片网格、表格行、任何有拖拽（react-dnd / dnd-kit）的列表。

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 统一时长 | `350ms` | 每个元素的动画时长 |
| 交错延迟 | `100ms` | 相邻元素的播放间隔 |
| 缓动曲线 | `ease-out` | 先快后慢 |

**keyframes**：

```css
@keyframes widgetFadeUp {
  0%   { opacity: 0; }
  100% { opacity: 1; }
}
```

**使用方式**：

```css
.your-list > * {
  animation: widgetFadeUp 350ms ease-out both;
}

.your-list > *:nth-child(1) { animation-delay: 0ms; }
.your-list > *:nth-child(2) { animation-delay: 100ms; }
.your-list > *:nth-child(3) { animation-delay: 200ms; }
/* ... */
```

**注意事项（关键）**：
- **`animation-fill-mode: both` 必须使用**：`backwards` 填充使元素在动画开始前保持 `opacity: 0`，避免闪烁；`forwards` 使元素在动画结束后保持 `opacity: 1`
- **不要在 keyframes 中加 `transform`**：react-dnd、dnd-kit 等拖拽库通过 inline style 设置 `transform` 控制拖拽位置。如果动画 keyframes 包含 `transform: translateY(...)`，结束后的填充值会覆盖拖拽库的 transform，导致拖拽无动画
- 同样适用：任何通过 JS 控制 `transform` 的场景（滚动视差、缩放拖拽等）

---

### 1.3 路由渐入（routeFadeIn）

**效果**：路由切换时页面内容从下方淡入。

**适用场景**：所有路由切换（包裹 `<Outlet />`）。

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 时长 | `300ms` | 单次路由切换动画时长 |
| 缓动曲线 | `ease-out` | |
| 垂直偏移 | `8px` | `translateY(8px)` |

**keyframes**：

```css
@keyframes routeFadeIn {
  0%   { opacity: 0; transform: translateY(8px); }
  100% { opacity: 1; transform: translateY(0);   }
}
```

**使用方式**：

```css
.transitionWrapper {
  animation: routeFadeIn 300ms ease-out;
}
```

```tsx
<div key={location.pathname} className={styles.transitionWrapper}>
  <Outlet />
</div>
```

**注意事项**：
- 必须用 `key={location.pathname}` 包裹 `<Outlet />`，否则 React 不重新挂载，动画不触发
- 适合页面级切换；组件内的小范围切换（如 Tab 切换）不适合，会因 key 变化重建所有状态
- **不与同时使用 `transform` 的 JS 库共用**：动画结束后 `transform: translateY(0)` 的填充值会覆盖 JS 设置的 transform（同 widgetFadeUp 的陷阱）。仅在纯页面过渡场景使用

### 1.4 登录卡片放大淡出（loginCardOut）

**效果**：登录卡片放大并淡出，背景同步消失，为工作台入场做铺垫。

**适用场景**：登录成功后跳转前、任何"完成操作→跳转新页面"的过渡场景。

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 时长 | `400ms` | 卡片放大淡出时长 |
| 缓动曲线 | `ease-in` | 先慢后快，出场自然 |
| 缩放倍数 | `1.05` | `scale(1) → scale(1.05)`，轻微放大 |
| 容器淡出 | `400ms ease-out` | 背景同步淡出 |

**keyframes**：

```css
/* 卡片放大淡出 */
@keyframes loginCardOut {
  0%   { opacity: 1; transform: scale(1);   }
  100% { opacity: 0; transform: scale(1.05); }
}

/* 容器背景淡出 */
.containerLeaving {
  transition: opacity 400ms ease-out;
  opacity: 0;
}
```

**使用方式**：

```tsx
const [transitioning, setTransitioning] = useState(false);

const handleSubmit = async (values) => {
  // ... login API call ...
  if (success) {
    setTransitioning(true);
    setTimeout(() => navigate('/'), 400);
  }
};

<div className={`${styles.container} ${transitioning ? styles.containerLeaving : ''}`}>
  <Card className={`${styles.card} ${transitioning ? styles.cardLeaving : ''}`}>
    {/* form */}
  </Card>
</div>
```

**注意事项**：
- `setTimeout` 延迟必须 ≥ 动画时长（400ms），否则页面可能在动画播完前就跳转了
- 容器和卡片使用不同的缓动（`ease-out` vs `ease-in`），产生层次感
- 卡片 `transform: scale(1.05)` 在 keyframes 中定义，动画结束后不保留（无 fill-mode），不影响后续渲染

### 1.5 工作台浮入（appFloatIn）

**效果**：整个工作台界面从下方浮入，与登录卡片的放大淡出衔接形成完整的登录过渡。

**适用场景**：登录后工作台首次挂载、任何整页级别的入场场景。

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 时长 | `500ms` | 浮入动画时长 |
| 缓动曲线 | `ease-out` | 先快后慢，入场自然 |
| 垂直偏移 | `24px` | `translateY(24px)`，从下方浮入 |
| 触发延迟 | `100ms` | 组件挂载后延迟触发，确保 DOM 就绪 |

**keyframes**：

```css
@keyframes appFloatIn {
  0%   { opacity: 0; transform: translateY(24px); }
  100% { opacity: 1; transform: translateY(0);     }
}
```

**使用方式（CSS Modules）**：

```css
/* 初始状态：隐藏 */
.appLayout {
  opacity: 0;
}

/* 触发类添加后播放 */
.appEntered {
  animation: appFloatIn 500ms ease-out forwards;
}
```

```tsx
const [entered, setEntered] = useState(false);

useEffect(() => {
  const timer = setTimeout(() => setEntered(true), 100);
  return () => clearTimeout(timer);
}, []);

<Layout className={`${styles.appLayout}${entered ? ` ${styles.appEntered}` : ''}`}>
  <Header>...</Header>
  <Content>...</Content>
</Layout>
```

**注意事项**：
- `forwards` 填充模式确保动画结束后工作台保持可见，不回弹
- 与 `menuItemFlyIn` 共享同一个 `entered` 状态（均 100ms 后触发），使侧边栏飞入和工作台浮入同时进行
- 使用 CSS Modules 局部类名（`.appLayout` / `.appEntered`），不与全局样式冲突
- **首次挂载只触发一次**：`useEffect([], [])` 确保仅登录后首次进入工作台时播放

### 1.6 登录→工作台完整过渡流程

**时序**：

```
用户点击登录
    │
    ├─ 登录中（button loading）
    │
    ├─ 登录成功
    │   ├─ setTransitioning(true)
    │   │   ├─ 卡片放大淡出 400ms (loginCardOut)
    │   │   └─ 背景淡出 400ms (containerLeaving)
    │   │
    │   └─ setTimeout 400ms → navigate('/')
    │
    ├─ MainLayout 挂载
    │   └─ useEffect 100ms → setSidebarEntered(true)
    │       ├─ 侧边栏逐项飞入 (menuItemFlyIn, 60ms 交错)
    │       └─ 工作台浮入 (appFloatIn, 500ms)
    │
    └─ 过渡完成，用户进入工作台
```

**协调要点**：
- 登录页的 `setTimeout(400ms)` 与 `loginCardOut` 时长一致
- 工作台的 `useEffect(100ms)` 给 React 渲染一帧缓冲区
- 两个组件不直接通信，通过路由切换自然衔接
- 总过渡时间 ≈ 400ms（登录）+ 100ms（缓冲）+ 500ms（工作台）= ~1s

## 2 动画参数速查

| 动画名 | 时长 | 缓动 | 交错延迟 | fill-mode | 包含 transform | 场景 |
|--------|------|------|---------|-----------|---------------|------|
| `menuItemFlyIn` | 300ms | ease-out | 40ms | forwards | 是 | 侧边栏入场 |
| `widgetFadeUp` | 350ms | ease-out | 100ms | **both** | 否 | 拖拽列表入场 |
| `routeFadeIn` | 300ms | ease-out | 无 | 无 | 是 | 路由切换 |
| `loginCardOut` | 400ms | ease-in | 无 | 无 | 是 | 登录卡片退出 |
| `appFloatIn` | 500ms | ease-out | 无 | forwards | 是 | 工作台整页入场 |

---

## 3 使用原则

1. **优先 GPU 合成属性**：只动画 `opacity` 和 `transform`，避免 `width/height/left/top/margin/padding` 等触发重排的属性
2. **拖拽场景只用 opacity**：任何有 `transform` 动画的元素不能同时被 JS 控制 `transform`（react-dnd 等）。拖拽列表只用 `widgetFadeUp` 模式
3. **首屏动画只播一次**：入场动画只应在用户首次看到时播放，路由切换、Tab 切换等不应重复触发
4. **`both` vs `forwards`**：
   - 需要防闪烁 → `both`（backwards 填充在延迟期间保持初始透明度）
   - 无延迟或不需要防闪烁 → `forwards`
5. **交错延迟计算**：延迟 ≈ 总时长 / 元素数 × 0.6（确保最后一个元素在总时长内播完）

## 4 新增动画流程

向本动画库新增动画时，需提供：

1. **效果描述**：一句话说明视觉效果
2. **适用场景**：明确适合和不适合的场景
3. **参数表**：常用可调参数列在表中
4. **keyframes 代码**：完整的 `@keyframes` 定义
5. **使用示例**：CSS + React 代码片段，可直接复制使用
6. **注意事项**：列出所有已知的冲突场景和限制（特别是与 JS 动画库的兼容性）
7. **更新动画参数速查表**（第 2 节）
