# 《统一工作台》开发基准文档 v2.0（最终版）

------

## 1. 项目概述

| 项目         | 内容                                                         |
| ------------ | ------------------------------------------------------------ |
| **名称**     | 统一工作台 (Unified Workbench)                               |
| **定位**     | 面向 20 人小团队的内网一体化协作与信息管理平台               |
| **核心模块** | 文件管理、内容管理、项目立项、模板系统、密钥记录、提醒推送、审计日志、备份 |
| **用户规模** | 20 人，年增量 < 10万条记录 / < 500GB 文件                    |
| **访问方式** | 内网为主，支持内网穿透；仅 PC 网页端（预留移动端扩展）       |

**设计理念：**

- **一站式**：团队所有信息集中管理
- **默认私有，自主公开**：用户掌控自己内容的可见性
- **模板驱动**：结构化数据录入，灵活自定义
- **轻量可控**：简易部署、安全可审计、所有数据位置可指定

------

## 2. 技术架构基准

### 2.1 技术栈（最终确定）

| 层级         | 技术                                                  |
| ------------ | ----------------------------------------------------- |
| 后端         | Python 3.11+ / FastAPI / SQLAlchemy 2.0 / Alembic     |
| 前端         | React 18 / TypeScript 5 / Ant Design 5                |
| 富文本编辑器 | Tiptap（仅启用：字号、颜色、加粗、下划线、斜体）      |
| 数据库       | PostgreSQL 15+                                        |
| 缓存/队列    | Redis 7                                               |
| 异步任务     | Celery（提醒推送、定时备份、文件导出）                |
| 实时推送     | WebSocket（站内弹窗）                                 |
| 部署         | Windows + Docker Desktop + Docker Compose             |
| 反向代理     | Nginx                                                 |
| 文档导出     | python-docx(Word) / WeasyPrint(PDF) / openpyxl(Excel) |

### 2.2 部署与存储架构

```text
[浏览器] ──HTTPS──> [Nginx] ──> [React 前端]
                       │
                       └─/api──> [FastAPI] ←──> [Redis]
                                    │              │
                                    │         [Celery Worker]
                                    ▼
                              [PostgreSQL]
─────────────────────────────────────────────────────
存储分工（路径在 docker-compose.yml 集中配置）：
  本地SSD（如 D:/Workbench/db_data）
    └─ PostgreSQL 实时数据库文件
  NAS（你指定目录）
    ├─ files/    ← 所有文件、图片（实时）
    └─ backups/  ← 数据库定时备份（频率可调）
```

**部署方式：** 安装 Docker Desktop → 配置 `.env` 路径 → 运行 `start.bat` 一键启动全部服务。

## 3. 功能模块基准

### 3.1 用户与权限模块

**用户管理：**

- 管理员创建账号，无开放注册
- 字段：用户名、密码(bcrypt)、昵称、头像、身份标签、状态(启用/禁用)
- 密码策略：≥8位，含字母+数字
- 认证：账号密码 + JWT；预留 2FA 接口（一期不实现）

**权限模型（管理员 / 普通成员 + 身份标签）：**

| 能力                                          | 管理员 | 普通成员          |
| --------------------------------------------- | ------ | ----------------- |
| 修改团队成员信息（增删用户、改他人资料/标签） | ✅      | ❌                 |
| 系统配置（推送渠道、备份设置）                | ✅      | ❌                 |
| 查看全部审计日志                              | ✅      | ❌（仅看自己相关） |
| 创建/管理自己的内容、文件、项目、记录         | ✅      | ✅                 |
| 访问公开内容                                  | ✅      | ✅                 |
| 创建/编辑模板                                 | ✅      | ✅                 |

**身份标签：** 自定义标签（如"研发""设计"），用于内容可见范围控制。

**内容可见性三态：**

- `私有`：仅创建者可见
- `公开`：所有登录用户可见
- `指定`：指定用户 / 指定身份标签可见

**强制规则：**

- 密钥/数据记录分区 → 强制私有
- 系统可配置某些分区强制公开（如公告区）

------

### 3.2 文件与图片模块

- 图片：单张 ≤ 20MB（jpg/png/gif/webp），自动生成缩略图
- 文件：单个 ≤ 100MB，任意类型
- 物理文件存 NAS，数据库仅存元数据（路径、大小、SHA-256、上传者、可见性）
- 上传计算哈希，用于去重和完整性校验
- 支持文件夹/标签分类
- 每个文件独立设置可见性
- 上传支持进度显示

------

### 3.3 内容模块

- 富文本内容发布（轻富文本：字号/颜色/加粗/下划线/斜体）
- 可附带文件、图片
- 独立可见性、标签、分类
- 分区域搜索（标题/标签/内容关键词）

------

### 3.4 项目 / 立项模块

- 基于模板填写表单方式创建项目
- 状态流转：`草稿 → 进行中 → 已完成 → 已归档`
- 项目内可关联文件、内容、提醒
- 项目可见性遵循三态规则
- 项目记录可导出 Word / PDF / Excel

------

### 3.5 模板系统模块 ⭐（核心）

**字段类型：**

| 类型        | 说明                                  |
| ----------- | ------------------------------------- |
| 单行文本    | 短文本                                |
| 多行文本    | 普通长文本                            |
| 轻富文本    | 字号/颜色/加粗/下划线/斜体（仅这5种） |
| 数字        | 整数/小数，可设范围                   |
| 日期/时间   | 日期选择                              |
| 单选下拉    | 预设选项选一                          |
| 多选        | 选多个                                |
| 是/否开关   | 布尔                                  |
| 文件上传    | 附件                                  |
| 图片上传    | 配图                                  |
| 标题/分割线 | 排版分组（非数据）                    |

**每个字段可配置：** 字段名、是否必填、默认值、提示文字、排序。

**模板管理（全程网页操作）：**

- 拖拽添加/删除/排序字段
- 模板分类（立项类、记录类、登记类等，自定义）
- 模板导入/导出（JSON 格式）
- 预置常用模板（开箱即用）

**模板版本策略（方案①，已确认）：**

> 模板修改后，**已填写的旧记录保留填写时的模板快照，保持原样显示**；新记录使用新版模板。

**实现方式：** PostgreSQL JSONB 存模板定义与填写数据；每条记录冗余存一份当时的模板 schema 快照。

**记录导出（已确认）：** 每条记录可随时导出为 Word / PDF / Excel。

------

### 3.6 密钥 / 数据记录模块 🔒

- 用于**记录（非生成）**API密钥、账号、配置等
- 字段：名称、类型、键值对内容、备注
- **强制 AES-256-GCM 加密存储**
- **强制私有**
- 密钥值默认打码，查看/复制时**二次输入登录密码验证**
- 分区域搜索（仅搜名称/备注，不搜加密值）

------

### 3.7 提醒与推送模块

**触发类型：**

- **定时提醒**：用户设定时间触发
- **事件触发**：
    - 高危内容操作（如删除文件、查看/修改密钥）→ 通知相关人/管理员
    - 他人指定提醒（@某人 / 指定接收人）

**推送渠道（按优先级，Celery 异步，插件化设计）：**

| 优先级 | 渠道          | 实现                   |
| ------ | ------------- | ---------------------- |
| 1      | 站内弹窗      | WebSocket 实时（必做） |
| 2      | 飞书 / 钉钉   | Webhook 机器人         |
| 3      | 企业微信 / QQ | 第三方API（后置）      |
| 4      | 邮箱          | SMTP（兜底）           |

> 每个渠道实现统一接口，可独立开关与配置。

------

### 3.8 审计日志模块

- 记录关键操作：登录、上传、删除、修改、权限变更、查看密钥等
- 字段：操作人、时间、操作类型、操作对象、IP、变更前后(diff)
- **只增不改不删**
- 管理员查看全部，普通成员查看自身相关
- 按人/时间/类型筛选

------

### 3.9 备份模块

- 管理员配置：备份目标路径（NAS）、频率（每日/每周等）、最大保留份数
- Celery 定时任务：数据库 dump 至 NAS `backups/`
- 文件本身已实时在 NAS，无需额外备份
- 超出保留份数自动清理最旧备份

## 4. 安全基准

| 项目     | 标准                                                         |
| -------- | ------------------------------------------------------------ |
| 传输     | 全站 HTTPS（内网自签 / 穿透层 TLS）                          |
| 密码     | bcrypt 哈希                                                  |
| 密钥分区 | AES-256-GCM 加密（方案A：主密钥服务器持有，配置文件妥善保管） |
| 查看密钥 | 二次输入登录密码验证                                         |
| 会话     | JWT + 刷新机制 + 闲置超时登出                                |
| 防护     | 登录失败限制、ORM 防注入、前端转义防 XSS                     |
| 文件     | 类型校验、重命名存储、禁止可执行解析                         |

## 5. 开发规范基准

**代码规范：**

- 后端：PEP8 + `ruff` + `black` + `mypy`
- 前端：ESLint + Prettier + 严格 TypeScript
- 所有 API 用 Pydantic 模型，自动生成 OpenAPI 文档

**后端项目结构：**

```text
backend/app/
├── api/        # 路由
├── core/       # 配置、安全、依赖
├── models/     # 数据库模型
├── schemas/    # Pydantic 模型
├── services/   # 业务逻辑
├── tasks/      # Celery 任务
└── utils/      # 工具
```

**API 规范：** RESTful；统一响应 `{ "code": 0, "msg": "", "data": {} }`

## 7. 数据库表结构设计（核心表）

```sql
-- 用户表
user
  id, username, password_hash, nickname, avatar,
  role(admin/member), status, created_at

-- 身份标签表
tag
  id, name, color, created_at

-- 用户-标签关联
user_tag
  user_id, tag_id

-- 可见性配置（通用，被多模块引用）
-- visibility 字段直接存于各内容表：
--   visibility: private/public/restricted
--   restricted_users(JSONB): 指定用户id列表
--   restricted_tags(JSONB): 指定标签id列表

-- 文件表
file
  id, name, stored_path, size, sha256, mime_type,
  folder_id, owner_id, visibility, restricted_users,
  restricted_tags, created_at

-- 文件夹表
folder
  id, name, parent_id, owner_id, created_at

-- 内容表
content
  id, title, body(JSONB富文本), owner_id, visibility,
  restricted_users, restricted_tags, tags(JSONB), created_at, updated_at

-- 模板表
template
  id, name, category, schema(JSONB字段定义),
  version, owner_id, created_at, updated_at

-- 记录表（含项目立项）
record
  id, template_id, template_snapshot(JSONB当时模板快照),
  data(JSONB填写内容), type(project/record),
  status(draft/ongoing/done/archived),
  owner_id, visibility, restricted_users, restricted_tags,
  created_at, updated_at

-- 密钥/数据记录表
secret
  id, name, secret_type, encrypted_data(加密), note,
  owner_id, created_at, updated_at

-- 提醒表
reminder
  id, title, content, trigger_type(timed/event),
  trigger_time, target_users(JSONB), channels(JSONB),
  status, creator_id, created_at

-- 审计日志表
audit_log
  id, user_id, action, target_type, target_id,
  detail(JSONB变更diff), ip, created_at

-- 系统配置表
system_config
  key, value(JSONB)   -- 推送渠道、备份设置等
```

## 8. 开发阶段规划

| 阶段        | 内容                                                         | 产出           |
| ----------- | ------------------------------------------------------------ | -------------- |
| **P0 地基** | Docker环境、数据库设计、用户认证、权限框架、整体布局         | 可登录的空系统 |
| **P1 核心** | 文件/图片/内容管理 + 可见性控制 + 审计日志 + 分区搜索        | 基础可用       |
| **P2 特色** | 模板系统(字段+轻富文本+版本快照) + 项目立项 + 密钥加密分区 + 记录导出 | 核心特色完成   |
| **P3 提醒** | 定时提醒 + 站内弹窗 + 飞书/钉钉推送 + 高危事件触发           | 提醒闭环       |
| **P4 收尾** | 邮件/企微推送 + 备份模块 + 性能优化 + 移动端适配底子         | 完整交付       |
